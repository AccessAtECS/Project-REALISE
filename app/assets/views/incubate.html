  3. Which licence to choose - Make sure you or your developer have this in mind early on as you will need to show this in the heading of your code e.g ATKit.js
  4. Tracking Development - Start this at the outset of the development e.g. ATKit tracking
  5. Where to keep the code - There are several repositories but the main ones are Source Forge and Github. You will need to register if you wish to add comments etc e.g. ATKit issues
